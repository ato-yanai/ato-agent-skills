## CP-005 一覧フィルタ機構

### 仕様

- クエリパラメータ規約: `?category=`（単一）/ `?tag=`（複数可・カンマ区切り）/ `?from=&to=`（日付範囲）/ `?sort=`（`new`|`old`）
- filters 組み立て: **複数軸は AND**、**同一軸の複数値は OR**。`-`/空/不正値は無視して全件にフォールバック。
- orders 変換: `sort=new` → `-publishedAt`、`sort=old` → `publishedAt`。
- 使える軸は画面ごとに異なる（各ページ設計書の「軸マトリクス」が正。News=カテゴリ、Blog=カテゴリ/タグ/日付）。
- UI: 適用・クリア、絞り込み0件時の表示はページ設計書の状態網羅に従う。

### 受け入れ基準(AC)

| AC | 内容 |
| --- | --- |
| AC-021 | `?category={id}` で該当カテゴリのみ表示。0件なら0件表示＋解除導線 |
| AC-022 | 不正な category 値は無視し全件表示 |
| AC-141 | Blog で `?tag=a,b` は a または b を含む記事を表示（OR） |
| AC-142 | Blog で `?from=2026-01-01&to=2026-03-31` は範囲内の記事のみ表示 |

## CP-006 ページネーション機構

### 仕様

- URL: `?page=N`（1始まり、省略時 1）。1ページ件数は各画面が指定（既定 12）。
- offset 計算: `offset = (N-1) * limit`。
- 境界: 1ページ目は「前へ」非活性、最終ページは「次へ」非活性。総ページ数 = `ceil(総件数 / limit)`。
- **範囲外（N>総ページ数 / N<1 / 非数値）: 404 を返す**（SEO上の重複・空ページを避ける）。

### 受け入れ基準(AC)

| AC | 内容 |
| --- | --- |
| AC-024 | `?page=2` で 13〜24件目を表示 |
| AC-025 | 最終ページで「次へ」が非活性 |
| AC-026 | `?page=999`（範囲外）/ `?page=abc` は 404 |

## CP-007 一覧META生成（ページNo付与）

### 仕様

- title: `{画面タイトル} | {サイト名}`。`N>=2` のとき末尾に `（Nページ目）` を付与。
- canonical: `?page=N` を含む正規URL（1ページ目は `page` を含めない）。
- prev/next: 前後ページがあれば `rel=prev` / `rel=next` を出力。
- 文言（画面タイトル・description）は各ページ設計書(SC)が指定し、組み立て規則のみ本機構が持つ。

### 受け入れ基準(AC)

| AC | 内容 |
| --- | --- |
| AC-027 | `?page=2` で title 末尾に `（2ページ目）` が付く |
| AC-028 | 1ページ目の canonical に `page` クエリが含まれない |
| AC-091 | 2ページ目以降も index、prev/next が正しく出力される |
