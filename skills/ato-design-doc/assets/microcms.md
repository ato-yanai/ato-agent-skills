### ロール定義

| ロール | 権限概要 |
| --- | --- |
| 管理者 | 全権限（サービス設定変更・メンバー管理含む） |
| 編集者 | コンテンツの作成・編集・公開。サービス設定変更不可 |
| カスタム | API 別・操作別（参照/下書き作成/公開/削除）を細かく設定 |

- 承認フロー利用時はカスタムロールで「申請者」（下書き作成+申請のみ）と「承認者」（承認+公開）を分ける。

### レビュー（承認フロー）

承認フローは API ごとに有効/無効を設定する。

**ステータス遷移:**

```
DRAFT（下書き） ──申請──▶ IN_REVIEW（申請中） ──承認──▶ PUBLISHED（公開）
        ◀──────却下────────────────────────────
PUBLISHED ──非公開──▶ CLOSED（非公開）
```

| ステータス | 誰が遷移 | 備考 |
| --- | --- | --- |
| DRAFT | 全ロール（作成時の初期状態） | プレビュー可 |
| IN_REVIEW | 申請者が「レビュー依頼」 | プレビュー可 |
| PUBLISHED | 承認者が「公開」または予約公開 | |
| CLOSED | 管理者・公開権限者が「非公開」 | コンテンツは削除されない |

### メディア管理

**許可形式（既定）:**

| 種別 | 形式 |
| --- | --- |
| 画像 | JPEG / PNG / GIF / WebP / AVIF / SVG |
| ファイル | PDF / Word (.docx) / Excel (.xlsx) / Zip 他 |

- アップロード上限サイズはプランにより異なる。フィールド単位の制限は各フィールドの制約列に記載。

**CDN URL 形式:**
```
https://images.microcms-assets.io/assets/{serviceId}/{fileId}/{filename}
```

**Image API 主要パラメータ:**

| パラメータ | 内容 | 例 |
| --- | --- | --- |
| `w` / `h` | 横幅 / 高さ（px） | `?w=800&h=600` |
| `fit` | リサイズ方法 | `contain` / `cover` / `fill` |
| `q` | 品質（1〜100） | `?q=75` |
| `fm` | 出力フォーマット | `webp` / `png` / `jpeg` / `avif` |
| `dpr` | デバイスピクセル比 | `?dpr=2` |

> Image API パラメータはフロントの `<Image>` コンポーネントや next/image の `loader` に渡す。ato-microcms-fetch の実装を参照。

## CMS-002 `news`（リスト形式）

### フィールド定義

> kind / 型は ato-microcms-types のマッピングに合わせる。「絞り込み/ソート」列で一覧機能の軸を確定する。

| fieldId | 表示名 | kind | TS型 | 必須 | 制約 | 絞り込み | ソート | 説明 |
| --- | --- | --- | --- | --- | --- | --- | --- | --- |
| title | タイトル | text | string | ✓ | 60字以内 | – | – | |
| category | カテゴリ | relation | `MicroCMSListContent<NewsCategory>` | ✓ | – | ✓ | – | depth=1 で展開 |
| publishedAt | 公開日 | date | string | ✓ | ISO 8601 | – | ✓ | |
| thumbnail | サムネイル | media | `MicroCMSImage` | – | – | – | – | |
| body | 本文 | richEditorV2 | string | ✓ | – | – | – | HTML文字列 |

#### `body` リッチエディタ設定

> microCMS 管理画面のフィールド設定で下表を参照してツールバーを ON/OFF する。フロントの sanitize-html 設定はこの表と一致させる（COMMON-009）。

| ツールバー機能 | 有効 | 出力タグ | 出力属性 |
| --- | --- | --- | --- |
| 段落 | ✓（固定） | `p` | `class` |
| 見出し H2 | ✓ | `h2` | `class`, `id` |
| 見出し H3 | ✓ | `h3` | `class`, `id` |
| 見出し H4 | – | `h4` | — |
| 太字 | ✓ | `strong` | — |
| 斜体 | ✓ | `em` | — |
| 下線 | – | `u` | — |
| 打ち消し線 | – | `s` | — |
| インラインコード | ✓ | `code` | `class` |
| リンク | ✓ | `a` | `href`（http/https/mailto のみ）, `target`, `rel` |
| 画像 | ✓ | `img` | `src`（CDN オリジン のみ）, `alt`, `width`, `height` |
| 動画埋め込み | – | `iframe` | `src`（許可ドメイン指定必須）, `width`, `height`, `allowfullscreen`, `loading` |
| 箇条書き | ✓ | `ul`, `li` | `class` |
| 番号付きリスト | ✓ | `ol`, `li` | `class` |
| 引用 | ✓ | `blockquote` | `class` |
| コードブロック | ✓ | `pre`, `code` | `class` |
| テーブル | – | `table`, `thead`, `tbody`, `tr`, `th`, `td` | `class` |
| 水平線 | – | `hr` | — |
| カスタムクラス | ✓ | （各タグに付与） | `class`（下表 allowlist のみ） |

**カスタムクラス設定:**

| クラス名 | 付与できる要素 | 用途 |
| --- | --- | --- |
| `note` | `p`, `blockquote` | 補足ボックス |
| `warning` | `p`, `blockquote` | 注意ボックス |

> 必要なクラスをここに追記し、対応するスタイルをデザインシステム側に定義する。`class` allowlist 方式は COMMON-009 のセキュリティ不変条件により必須。

---

- 共通フィールド（`id`/`createdAt`/`updatedAt`/`publishedAt`/`revisedAt`）は取得ラッパーが付与（型に含めない）。
- **制約列** ＝ 入稿時バリデーション（文字数・形式・一意性など。microCMS のフィールド設定に対応）。**microCMS の機能で担保されるため、ここでは AC を設けない**（サイト側の挙動や入力フォームの検証は page / COMMON-010 の AC が持つ）。
- **一意性**が必要なフィールド（slug 等）は microCMS の「重複を許可しない」を有効化し、制約列に「一意」と記す（本例は `[id]`＝コンテンツIDを使うため slug を持たない）。
- **メディア（画像・ファイル）の形式・サイズは既定で制限しない。** 要件があるフィールドだけ制約列に推奨/最大サイズ・許可形式を書く（例: `推奨1200×630・JPG/PNG/WebP・2MBまで`）。
- **richEditorV2 フィールド**はフィールド定義表の直下に「リッチエディタ設定」節を設け、有効ツールバー・出力タグ・カスタムクラスを定義する。フロントのサニタイズ規約は COMMON-009。
- 各画面の**取得条件**（filters/orders/limit/offset/depth）はページ設計書の「データ取得」節が持つ（変換規約は COMMON-005）。

### 権限・公開ワークフロー

| ロール | 権限 | 備考 |
| --- | --- | --- |
| 編集者 | DRAFT 作成・IN_REVIEW 申請 | PUBLISHED 操作不可 |
| 管理者 | PUBLISHED 公開・予約公開・CLOSED 非公開・削除 | |

- 公開フロー（承認フロー有効時）: `DRAFT → IN_REVIEW → PUBLISHED`（予約公開可）。却下時は DRAFT に戻る。
- 承認フロー無効時: `DRAFT → PUBLISHED` の直接公開（ロール権限があれば誰でも公開可）。
- **予約公開**: 公開日時を未来に指定して自動公開。**公開停止**はステータスを CLOSED に変更（コンテンツは削除しない）。
- ロールの全体定義・APIキー・承認フロー設定・メディア管理は「サービス設定」を参照。

### Webhook / 再検証

| トリガー | 対象 | 動作 |
| --- | --- | --- |
| `news` の公開/更新/削除 | `/news`・`/news/[id]`・TOP | revalidateTag（署名検証は ato-microcms-webhook） |

- 再検証タグの規約は ato-microcms-fetch のタグ規約に合わせる。
