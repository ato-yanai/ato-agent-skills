# レビュー・監査モード（出口ゲート）

既存の設計書を受け取り、抜け漏れを検出して指摘するモード。前提は SKILL.md の「方針」「設計の流れと前後関係」「共通規約」「設計の原則」。
属人化した「見落とし」を構造的に潰すのが目的。

**このモードは「実装・テストへ渡せるか」を判定する出口ゲート**（create の入力ゲートと対になる）。
下のチェックリストが go/no-go の合否表で、全項目が通れば実装・テストへ進められる。
通らない項目は「実装・テストに渡せない箇所」として必須修正に挙げる。

## 手順

1. **対象を読む。** レビューする設計書（および関連する親＝機能一覧・サイト全体）を読む。
2. **チェックリストで突く。** 下のリストを順に当て、欠落・矛盾・規約違反を洗い出す。
3. **指摘を出す。** 各指摘は **「箇所（ID/章）→ 何が欠けているか → どう直すか」** の順で具体的に、日本語で書く。
   重大度（必須修正／推奨／軽微）を付け、必須修正を先頭に並べる。
4. **go/no-go を判定する。** 件数サマリと、**実装・テストへ進めるか（必須修正が残っていないか）**を最後に明言する。
   修正まで求められたら作成モードの要領で直す。

## チェックリスト（＝実装・テストへ渡す合否表）

### A. トレーサビリティ（実装に渡せるか）

- [ ] **網羅**：機能一覧(FUNC)に全機能が載っているか。実装/画面にあって FUNC に無い機能はないか。
- [ ] **トレーサビリティ**：各 FUNC に関連 PAGE/CMS/COMMON/AC が埋まっているか。**存在しない ID への参照（リンク切れ）**は無いか。
- [ ] **重複行**：同名機能が複数行に分かれていないか（置き場所違いは関連画面列で表現すべき）。
- [ ] **共通化**：絞り込み/ページャ/META 等の機構が各ページに重複記述されていないか（COMMON に集約すべき）。
- [ ] **設計の原則違反**：別ドメインの同種機能を1行に潰していないか／META・ページNo付与が2か所の責務を混在させていないか。

### B. テスト基盤（テストに渡せるか）

- [ ] **受け入れ基準**：各仕様に AC があるか。**AC 空欄＝未設計**として指摘する。曖昧語（適切に/正しく）で逃げていないか。
- [ ] **AC の粒度**：入力と期待結果が書かれ、**そのままテストケースになる**か。
- [ ] **状態網羅**：一覧/フォーム系で「0件・絞り込み0件・エラー・ローディング・権限なし・範囲外・レスポンシブ」が漏れていないか。
- [ ] **非機能**：性能/SEO/アクセシビリティ/対応環境/セキュリティが宣言され、AC に落ちているか。

### C. データ・画面の確定

- [ ] **microCMS**：フィールド・TS型・必須・**絞り込み/ソート軸**・取得条件（filters/orders/limit/offset/depth）が確定しているか。
- [ ] **画面・URL**：URL設計・画面遷移が確定し、各 PAGE のデータソースが決まっているか。
- [ ] **ページ分割**：各画面に対応する `page-<slug>.md` があり、site.md 画面一覧の「詳細ファイル」列と一致するか（ファイル欠落・孤立ファイルが無いか）。

### D. 客先体裁・版

- [ ] **客先体裁**：変更履歴・前提/用語・図（サイトマップ/遷移）があるか。専門用語に偏っていないか。
- [ ] **版・文書管理**：提出済みなら各文書（ページは `page-<slug>` 単位）に版・Gitタグが入っているか。機能一覧の文書管理表（親・横断／ページ単位の2表）が最新か。

### E. 残課題

- [ ] **未確定の明示**：仕様未確定の箇所が「客先確認事項」として残課題に挙がっているか（空欄のまま紛れていないか）。

## 指摘の書き方（例）

> **[必須] PAGE-003 ニュース一覧／状態網羅**
> 「絞り込み結果0件」と「範囲外ページ（?page=999）」の状態が未記載。
> → 状態網羅表に2行追加し、それぞれ AC を参照/採番（例 AC-021 0件表示＋解除導線／AC-026 範囲外は404）。

> **[推奨] 機能一覧 FUNC-012 ソート／受け入れ基準**
> AC 列が空欄＝未設計。`?sort=` の値ごとの並び順を AC 化する（AC-023）。

## 総括の書き方（例）

> **判定：no-go（実装・テストへ進めない）**
> 必須修正 2 件 / 推奨 3 件 / 軽微 1 件。
> 必須の AC 空欄（FUNC-012・状態網羅2件）が残るためテストケースを導出できない。これらを潰せば go。
